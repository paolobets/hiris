## Come accedere a HIRIS

HIRIS espone due metodi di accesso:

| Metodo | URL | Quando usarlo |
|--------|-----|---------------|
| **Ingress HA** (raccomandato) | HA → Add-on → HIRIS → "Apri interfaccia web" | Uso normale, autenticato tramite HA |
| **Porta diretta** (testing) | `http://192.168.1.95:8099/` | Verifica rapida API, debug, sviluppo |

> ⚠️ La porta diretta è aperta nella fase sperimentale per facilitare i test. In produzione verrà chiusa.

**Per testare le API via porta diretta** (quando indicato nei passi):
```
http://192.168.1.95:8099/api/health
http://192.168.1.95:8099/api/status
```

**Per testare le API via Ingress** (dal browser con HIRIS già aperto): DevTools → Console:
```js
fetch('api/health').then(r => r.json()).then(console.log)
```

## Cosa è stato implementato in v0.0.2

- **HA Client** — REST + History API + WebSocket (state_changed)
- **8 strumenti Claude** — entity states, energy history, weather forecast, service call, notifiche, automazioni
- **Claude Runner** — agentic loop multi-turn con tool use (AsyncAnthropic)
- **AgentEngine** — scheduler APScheduler, trigger state_changed, CRUD agenti
- **API REST** — `api/chat`, `api/agents` (CRUD), `api/status`, `api/health`
- **Chat UI** (pannello principale) — interfaccia NL dark theme, storico conversazione, sidebar agenti
- **Agent Designer** (link "Configura agenti" nella sidebar) — form step-by-step, trigger condizionali, 8 checkbox strumenti
- **32 test automatici** — copertura ha_client, tools, claude_runner, agent_engine, API
