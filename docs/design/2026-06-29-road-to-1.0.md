# HIRIS — Road to 1.0 (rimozione flag `experimental`)

**Stato di partenza:** v0.19.0, `stage: experimental` in `hiris/config.yaml`.
**Tesi:** codice e feature sono già di livello 1.0 (731 test, CI, audit semaforo
superato, UI mobile a posto). Il gap per togliere `experimental` è ~80%
**operativo/validazione + sicurezza-dati + documentazione**, non sviluppo di feature.

Il 1.0 si dichiara quando TUTTI i blocker sotto sono ✅. Ogni item ha criterio di
accettazione verificabile.

---

## Blocker 1 — Soak / rodaggio reale  ⏳ (tempo)
Le capability costruite a giugno 2026 (storico, cattura `state_changed`, digest
notturno, `get_history` via MCP, apply reale proposte) sono verdi sui test ma non
ancora rodate sul campo.

- [ ] 0.19.0 in esecuzione reale su HA per **≥ 3–4 settimane** senza crash né
      perdita dati.
- [ ] Verificare almeno una volta in reale, end-to-end:
  - [ ] cattura: abilita una categoria in *Storicizzazione* → dopo qualche giorno
        `get_history` restituisce bucket reali.
  - [ ] digest: dopo ~1 settimana compaiono insight settimanali (recall_knowledge).
  - [ ] compact notturno gira senza errori (log) e il DB resta limitato.
  - [ ] proposta automazione: Claude propone → *Attiva* → l'automazione **esiste e
        gira** in HA.
  - [ ] semaforo: azione gialla → notifica iPhone → approva → esegue; rossa →
        conferma in HIRIS.
- [ ] Raccogliere i problemi emersi e chiuderli.

## Blocker 2 — Sicurezza dei dati negli upgrade  🟢 (v0.20.0, quasi完)
~~Gli store SQLite usano solo CREATE TABLE IF NOT EXISTS, nessuna migrazione.~~

## Cosa NON serve per 1.0 (evitare over-engineering)
Nuove feature; più test unitari fini a sé stessi; riscritture. Feature complete,
suite solida, CI presente, mobile a posto, semaforo ben progettato e hardenato.

## Sequenza consigliata
1. (fatto) 0.19.0 live + redeploy gateway.
2. **Blocker 2** (migrazioni DB) — codeabile subito, sblocca la sicurezza upgrade.
3. **Blocker 3 + 5** (security doc + pip-audit + doc) in parallelo al soak.
4. **Blocker 4** (gateway in CI/remote).
5. **Blocker 1** (soak ≥3–4 settimane) corre in parallelo a tutto.
6. Quando tutti ✅ → rimuovere `stage: experimental` → **1.0.0**.
